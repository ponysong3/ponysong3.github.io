基于Hexo 5.0搭建个人博客

Posted on 2022-03-22 Edited on 2022-07-02 In Hexo

Hexo 5.0做了较大更新，最主要的变化是hexo5.0和next支持源码与配置隔离，即在指定目录下新增配置文件，修改该配置文件即可覆盖源码里面的配置，这样可以有效避免修改源码，以后可以放心大胆升级hexo和next，而不用担心升级覆盖源码。

Hexo部署

  1. 登录VPS，新建用户pony，因为VPS主机刚开始只有root用户，而brew工具不能使用root安装

    useradd -m pony

    passwd pony #设置用户密码

  2. 安装brew工具，根据提示走，命令如下

    sh -c "$(curl -fsSL https://raw.githubusercontent.com/Linuxbrew/install/master/install.sh)"

  3. 添加brew的全局环境变量

    echo 'export PATH="/home/pony/.linuxbrew/Homebrew/bin:$PATH"' >> /etc/profile    #下面hexo也用这个环境变量

    source /etc/profile

  4. 安装nodejs长期支持版

    brew search node

    brew install node@12

    echo 'export PATH="/home/pony/.linuxbrew/opt/node@12/bin:$PATH"' >> /etc/profile    #node环境变量

    source /etc/profile

  5. 利用npm安装hexo

    npm install -g hexo-cli

    hexo init hexo

    cd hexo

    nom install

    hexo server

    hexo不需要单独设置环境变量了，跟brew共用一个环境变量

  6. 查看hexo版本

    hexo -v或hexo version

    要切换到普通用户查询，root用户查询不显示hexo版本只显示hexo-cli版本

  7. 在pony家目录下，创建hexo目录，初始化博客，生成博客相关的文件

    hexo init

  8. 启动博客，并根据提示检查http://[VPS IP]:4000是否打开正常

    hexo s #用于在本地测试，后续博客要部署到Github

    CnetOS 默认是不开放端口的，需要手动开放端口

    firewall-cmd --zone=public --add-port=4000/tcp --permanent firewall-cmd --reload

  9. 创建新博客

    hexo n "my_first_blog”   #新建博客

    vi ~/hexo/source/_posts/my_first_blog

  10. 重新渲染博客

    **hexo clean**  #清理老的渲染文件

    **hexo g** #生成博客，在public目录下生成新的博客渲染文件

    **hexo s** #启动博客

  11. 把博客部署到远端Github

    • 在github上创建仓库，仓库名必须是“用户名.github.io”
    • 在hexo目录下安装git部署插件npm install --save hexo-deployer-git （报警请忽略）

      git config --global [user.name](http://user.name/) “ponysong3” git config --global user.email “ponysong3@gmail.com”

    • 部署到远端Github

      hexo d

    • 后续的博客更新常用命令为
      hexo clean && hexo g && hexo d

  12. 升级Hexo版本

    npm update #没事千万别升级，升级之前千万要备份

      **categories**: /categories/ || fa fa-th

      **archives**: /archives/ || fa fa-archive

      #schedule: /schedule/ || fa fa-calendar

      #sitemap: /sitemap.xml || fa fa-sitemap

      #commonweal: /404/ || fa fa-heartbeat
